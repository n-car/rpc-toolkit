# Contributing

Issues and pull requests are welcome.

This repository is the ecosystem hub. Implementation-specific fixes should usually be submitted to the relevant implementation repository. Cross-runtime proposals, compatibility notes, documentation improvements, and ecosystem-level design discussions can be opened here.

## Compatibility Reports

When reporting interoperability behavior, include:

- implementation and runtime;
- version or commit;
- transport;
- Safe Mode enabled or disabled;
- strict mode state, if applicable;
- batch or notification usage;
- minimal request and response;
- expected behavior;
- actual behavior.

## Security And Credentials

Do not submit private credentials, private network details, customer names, or private project identifiers in issues, examples, logs, or pull requests.

Security issues should follow [SECURITY.md](SECURITY.md).

## Documentation Style

Keep documentation technical, concise, and public-facing. Avoid claims that are not backed by implementation documentation or tests.
